# Virtualization

虚拟化技术

  • Originally By mengmengqiang

# 1. 虚拟化介绍

最早出现在20世纪60年代的IBM大型机系统。 使用一种叫做虚拟机监控器的程序在物理硬件上生成可以独立运行操作系统软 件的虚拟机实例。 降低了IT成本，增强系统安全性和可靠性。

# 2. 虚拟化技术分类

虚拟化技术的分类有很多种，每一种的分类的不是绝对的，一个优秀 的虚拟化软件往往融合了多项技术。

# Platform Virtualization

平台虚拟化(Platform Virtualization) 是针对计算机和操作系统的虚拟化

平台虚拟化又可以细分为一下几个小类

# 全虚拟化

完全模拟底层硬件，使得为原是硬件设计的操作系统或者其他软件系统 完全不做任何修改就能使用，就是我们常用的虚拟机，使用为硬件平台 准备的安装镜像就能直接为虚拟机安装操作系统。

# 超虚拟化

使用特殊的方式减少了额外的异常中断和模拟部分硬件执行流程，以此 大幅度提升性能。著名的有Xen

# 硬件辅助虚拟化

指借助硬件的主持来实现高效的全虚拟化，比如Inter-VT技术， AMD-V技术。

# 部分虚拟化

(Full Virtualization) 通向全虚拟化的一个里程碑。

# 操作系统级虚拟化

(Operating System Level Virtualization) 著名的有OpenVZ等。

# Resource Virtualization

资源虚拟化(Resource Virtualization) 是针对特定的系统资源的虚拟化，比如内存、存储、网络资源等。

# Application Virtualization

应用程序虚拟化(Application Virtualization) 将应用程序与操作系统解耦合，为应用程序提供虚拟的运行环境，包括 应用程序的可执行文件和运行时环境。

和平台虚拟化相比，虚拟应用对资源依赖更小，在提供虚拟化优势的同时没有引入其他开销。部署更简单，更灵活，更细粒度的安全策略

# 3. 具体的虚拟化应用

# Container VS VM

容器 虚拟机
  • 容器在Linux上本机运行，并与其他容器共享主机内核。它运行一个独立的进程，不占用比任何其他可执行文件更多的内存，使它变得轻量级。
  • 相比之下，虚拟机(虚拟机)运行完整的“客户”操作系统，通过虚拟机管理程序虚拟访问主机资源。一般来说，虚拟机提供的环境资源比大多数应用程序需要的都多。

# KVM 虚拟化

  • Kernel based Virtualization

  • KVM 是一个完整的虚拟化解决方案，适用于包含虚拟化扩展（Intel VT或AMD-V）的x86硬件上的Linux。它由可加载的内核模块kvm.ko组成，它提供核心虚拟化基础架构和处理器特定模块，kvm-intel.ko或kvm-amd.ko。

  • 使用KVM，可以运行多个运行未修改的Linux或Windows映像的虚拟机。每个虚拟机都有专用的虚拟化硬件：网卡，磁盘，图形适配器等。

  • KVM是开源软件。KVM的内核组件包含在主线Linux中，截至2.6.20。KVM的用户空间组件包含在主线QEMU中，从1.3开始。

# 容器虚拟化

Docker 容器技术简介-Huoty's Blog

docker 应该说是当前应用较为广泛的容器技术之一，主要介绍这个

  • Docker是一个开源的应用容器引擎，它让开发者可以打包他们的应用以及依赖包到一个可移植的容器中，然后发布到安装了任何 Linux 发行版本的机器上。
  • Docker基于LXC来实现类似VM的功能，可以在更有限的硬件资源上提供给用户更多的计算资源。与同VM等虚拟化的方式不同，LXC不属于全虚拟化、部分 虚拟化或半虚拟化中的任何一个分类，而是一个操作系统级虚拟化。
  • Docker是直接运行在宿主操作系统之上的一个容器，使用沙箱机制完全虚拟出一个完整的操作空间，容器之间不会有任何接口，从而让容器与宿主机之间、容器与容器之间隔离的更加彻底。每个容器会有自己的权限管理，独立的网络与存储栈，及自己的资源管理,能使同一台宿主机上可以友好的共存多个容器。
  • Docker借助Linux的内核特性，如:控制组(Control Group)、命名空间(Namespace)等,并直接调用操作系统的系统调用接口。从而降低每个容器的系统开销，并实现降低容器复杂度、启动快、资源占用小等特征。即: docker是共享操作系统内核的。
