服务管理

# Linux服务管理

# 服务分类

Linux服务 RPM包默认
安装的服务		 独立的服务
基于xinetd服务
源码包安装的服务

其实之所以分成这样3类，要用不同的管理方法来管理，归根到底是服务的安装位置不同

  • 服务启动
  • 服务自启动

rpm安装服务和源码包安装服务的区别

  • 源码包安装在指定位置，一般是/usr/local
  • rpm包安装在默认位置中，在SysV服务管理系统中，启动脚本一般默认放在/etc/rc.d/init.d,service命令默认只能管理rpm安装的服务正是因为它默认搜索/etc/rc.d/init.d目录下的服务

# 查询已安装的服务

  • rpm包安装的服务：chkconfig --list可以看到所有rpm包服务的自启动状态

查看服务当前运行状态一般用ps -aux | grep,而netstat -tlun只能看到占用端口的服务状态

  • 源码包安装的服务：查看服务安装的位置，一般是/usr/local下

# RPM服务管理

  • rpm包服务安装的默认位置
位置 作用
/etc 配置文件位置
/etc/sysconfig 初始化环境配置文件位置
/etc/xineted.conf xinetd本身配置文件
/etc/init.d /etc/rc.d/init.d的软链接,(SysV服务管理系统中)独立服务启动脚本的位置
/etc/xineted.d 基于xinetd服务的启动脚本存放位置
/var/lib 服务产生的数据位置
/var/log 日志

除了这些目录，部分rpm包还会有自己特殊文件的保存目录，不如httpd会把网页文件存放在/var/www/html目录下

# 独立服务管理

# 启动

/etc/init.d/独立服务名 start|stop|status|restart

service 独立服务名 start|stop|restart|status

这两者本质上都是进入/etc/init.d/目录下寻找服务，然后对其操作 service命令是红帽为了简化操作专门开发出的专有命令

service --status-all列出SysV服务管理系统下的所有服务状态

# 自启动

  • chkconfig --level 2345 httpd on//默认级别就是2345，所以可以直接chkconfig httpd on

  • （建议）修改/etc/rc.d/rc.local文件（如果有）

在每次计算机启动完所有服务之后，在用户输入登录密码之前，系统会自动读取这个文件，执行里面的命令，所以只要把启动命令加入文件中就能实现开机自启动

  • 使用ntsysv命令管理自启动
  • 图形界面，给自启动的服务打*
  • 不仅可以自启动独立服务，还可以自启动基于xinetd服务
  • 红帽专有命令

# 基于xinetd服务管理

  • 描述：超级守护进程，由于该服务本身不安全，使用比较少，只有有部分不安全的服务依然基于xinetd。
  • xinetd本身是独立服务，存在于内存，但是其所管理的服务只有在xinetd响应之后才会运行，可以减少内存占用
  • 启动服务
vim /etc/xinitd.d/telnet


service telnet          # 服务名称
{
    flag    =REUSE      # 标记REUSE，设定TCP/IP socket可重用
    socket_type     =stream # 使用TCP协议数据包
    wait    =no # 允许多个连接同时进行
    user    =root   # 启动服务的用户
    server  =/usr/sbin/in.telnetd   # 服务的启动程序
    log_on_failaure +=USERID    # 登录失败后，记录用户的ID
    disable =no # 启动服务
}


service xinetd restart

# 源码包服务的管理

  • 启动

归根到底，还是调用启动骄傲本来启动。 不同源码包启动脚本不同，查看安装说明。

  • 自启动

修改/etc/rc.d/rc.local

  • 让源码包服务被服务管理命令识别(不建议 )
# 可以让 service 命令识别
ln -s Path/of/Service /etc/init.d/SeviceName

# 可以让 chkconfig和 ntsysv 命令识别
vim /etc/init.d/ServiceName

    #!/bin/sh
    #
    # chkconfig: 35 86 76
    # description: 运行级别 启动顺序 关闭顺序
    #上面 chkconfig 和 description 两行必须要有才会被chkconfig识别
    #启动顺序（S+num+service）和关闭顺序（K+num+service）不能与现有顺序重合

chkconfig --add ServiceName
chkconfig --del ServiceName
