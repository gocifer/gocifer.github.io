# 命令行

windows---用户优先 linux---服务器优先（稳定性，远程管理，减少资源占用）

# 解决问题的智慧

英文

帮助 示例 文档 查找

忘掉windows思维方式

# 虚拟机安装与使用

硬件管理 快照 克隆

# 磁盘分区

提升读取效率

# 分区

主分区：最多只能有4个

扩展分区：1个，加主分区最多四个，不能写入数据格式化，只能包含逻辑分区

逻辑分区：

# 格式化

  • 高级格式化---逻辑格式化 写入文件系统，在磁盘的拓尔思南岗区与写入特定数据，建立inodo列表。在分区中划出一片用于存放文件分配表、目录表等用于文件管理的磁盘空间

block-区块 i节点号-inode号

# 硬件设备文件名

linux中所有设备皆为文件

设备名 文件名
IDE /dev/hd[a-d]
SCSI/SATA/USB /dev/sd[a-p]
光驱 /dev/cdrom或/dev/sr0
软盘 /dev/fd[0-1]
打印机（25针） /dev/lp[0-2]
打印机（USB） /dev/usb/lp[0-2]
鼠标 /dev/mouse

# 分区文件名

/dev/sda1 逻辑分区自动从5开始识别

# 挂载

  • 必须分区
    • /
    • swap
  • 推荐分区
    • /boot

# 系统安装

# 主机名

windows:局域网内相同主机名不能互相通信 linux:没有此限制

# 密码原则

  • 复杂性
  • 易记忆性
  • 时效性

# 安装日志

  • /root/install.log:安装在系统中的软件包和版本信息
  • /root/install.log.syslog：安装过程中留下的事件记录
  • /root/anaconda-ks.cfg：以Kickstart配置文件的格式记录安装过程中设置的选项信息

# 远程登录管理工具

# 虚拟网卡配置

ifconfig ens32 192.168.1.102

# 远程连接工具

ssh username@hostnameorssh username@ip_address

# 远程数据传输

scp usera@host1:/from/file userb@host2:/to/

scp -r usera@host1:/from/directory userb@host2:/to/

# linux学习注意事项

# 给初学者的建议

  • linux区分大小写
  • linux中所有内容以文件形式保存
  • linux不靠扩展名区分文件类型（我们所见的扩展名仅是为了方便管理员区分）
  • linux所有存储设备都必须挂载之后用户才能使用
  • Windows/Linux程序不能通用

# 服务器管理和维护建议

1.linux各目录的作用

2.服务器注意事项

  • 远程服务器不允许关机，只能重启
  • 重启时应该关闭服务
  • 不要在服务器访问高峰运行高负载命令
  • 远程配置防火墙时不要把自己踢出服务器
  • 指定合理的密码规范并定期更新
  • 合理分配权限（够用就好）
  • 系统启动的服务越少越好
  • 定期备份重要数据和日志
