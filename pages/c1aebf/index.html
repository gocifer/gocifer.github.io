# Linux系统管理

# 进程管理

什么是进程？

  • 进程是正在执行的一个程序或命令，每一个进程都是是一个运行的实体，都有自己的地址空间，并且占用一定的系统资源。
  • 每一个程序的运行都会在系统中产生一个或多个进程
  • 即使是ls这样的简单命令也会产生进程，但是由于该命令持续时间非常短暂，所以不便监测。

进程管理的作用

  • 判断服务器健康状态
  • 查看系统中所有进程
  • 杀死进程

进程管理的工具

  • 对于规模较少的服务器，简单的系统管理命令就可以胜任
  • 但在大规模的服务器场景中，就需要使用专门的监控软件或搭建专门的监控服务器来较少运维工程师的工作量

# 进程查看

# PS---查看系统中所有进程

  • 组合1：ps aux
    • 描述：查看系统中所有进程，使用BSD操作系统格式
    • 选项：
      • a：显示前台进程
      • u：显示用户信息
      • x：显示后台进程
  • 组合2：ps -le
    • 描述：查看系统中所有进程，使用Linux操作系统格式
    • 选项：
      • -l：显示详细信息
      • -e：显示所有进程
  • 输出：
    • USER：由哪个用户产生
    • PID：进程编号，供计算机识别。(在SysV服务管理系统中正常的PID=1进程是/sbin/init；在systemd服务管理系统中则是/usr/lib/systemd/system)
    • %CPU：该进程占用CPU资源的百分比
    • %MEM：该进程占用物理内存的百分比
    • VSZ：该内存占用虚拟内存的大小(KB)
    • RSS：该进程占用实际物理内存的大小(KB)
    • TTY：该进程在哪个终端中运行的。其中tty1-7代表本地控制台终端，pts/0-255代表虚拟终端，？代表该进程直接由内核产生，而不是由终端产生
    • STAT： 进程状态。R--运行；S--睡眠；T--停止；s--包含子进程；+--位于后台；
    • START：进程启动时间
    • TIME：占用CPU计算时间
    • COMMAND：产生进程的命令名

# pstree---查看进程树

  • 格式：pstree [选项]
  • 选项：
    • -p：显示进程的PID(折叠的子进程也会展开)
    • -u：显示进程的所属用户

# top---查看系统健康状态

  • 格式：top [options]
  • 选项：
    • -d 秒数：指定几秒更新，默认为3
  • 交互模式选项：
    • h/?：显示帮助
    • P：默认选项，以CPU使用率排序
    • M：以内存使用率排序
    • N：以PID排序
    • q：退出top命令
  • 注意：top命令本身很占资源，不要总是或一直打开

# 终止进程

  • 一般系统的进程都有自己的终止方法，杀死进程是在正常方法无法生效时，才会使用的手段。
  • 对于非法进程（比如病毒、木马等）简单的杀死进程并不能彻底解决办法，应删除源文件或使用专业的杀毒软件。

# kill

  • kill -l
  • 描述：查看可用的进程信号
  • 常用信号：
    • 1：重启。让进程立即关闭，然后重新读取配置文件之后启动。
    • 2：终止前台进程。相当于C-c。
    • 8：发生致命的算术运算错误时发出。包括浮点运算错误，溢出及除数为0等其他所有的算术错误。
    • 9：强制终止。本信号不能被阻塞、处理、或忽略。
    • 15：正常结束进程的信号，kill命令的默认信号。有时如果进程已经发生问题该信号无法正常终止进程，应该用9。
    • 18：让暂停的进程恢复执行，本信号不能被阻断。
    • 19：暂停前台进程，相当于C-z,本信号不能被阻断。
  • 示例：
    • kill -1 12345
    • kill -9 12345

# killall

  • 格式：killall [选项][信号] 进程名
  • 描述：按照进程名杀死进程
  • 选项：
    • -i：交互式，询问是否要杀死某个进程
    • -I：忽略进程名的大小写
  • 示例：kill -I 9 chrome

# pkill

  • 格式：pkill [选项][信号] 进程名
  • 选项：
    • -t 终端号：按照终端号踢出用户
  • 示例：pkill -9 -t pts/1

# 工作管理

# 把进程放入后台

  • 方法1：
    • CommandLine &
    • 描述：放入后台继续运行，类似windows最小化
  • 方法2：
    • Ctrl-z
    • 描述：放入后台暂停运行，长时间不处理将把其从物理内存转移至虚拟内存。

# 查看后台工作

  • jobs [-l]
  • 选项：
    • -l：显示工作的PID
  • 注释
    • +：表示最近一个放入后台的工作，也是工作恢复时默认恢复的工作
    • -：倒数第二个放入后台的工作

# 恢复进程

  • bg %工作号把工作恢复到后台
  • fg %工作号把工作恢复到前台
  • 注意：

%号可以省略，PID和工作号区分 有些进程比如top``vim必须恢复到前台交互才能正常运行，否则不会恢复

  • 示例：
    • bg +
    • fg 12345

# 系统信息查看

# vmstat监控系统资源

  • 格式：vmstat refresh-timeout refresh-times

# dmesg开机时内核监测信息

是一个用来查看硬件信息的重要信息

# free命令查看内存使用状态

  • 缓存：用来加速数据的“读取”，先将数据读取到内存的cache区，再来进行快速的读取。
  • 缓冲：用来加速数据的“写入”，先将要保存的数据存到buffer缓冲区，满足一定条件后才会写入硬盘。
