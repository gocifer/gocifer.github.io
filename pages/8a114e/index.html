# 正则表达式与通配符

正则表达式:用来在文件中匹配符合条件的字符串，正则是包含匹配。grep、awk、sed等命令可以支持正则表达式。

通配符:用来匹配符合条件的文件名，通配符是完全匹配。ls、find、cp这些命令不支持正则表达，所以只能使用shell自己的通配符来进行匹配。

通配符 作用
？ 匹配任意一个字符
* 匹配任意内容（0或多个字符）
[ ] 匹配括号中任意一个字符，eg：[abc],[123]
[-] 匹配括号中范围内的字符
[^] 逻辑非，匹配非括号中的一个字符

# 基础正则表达式

元字符 作用 示例
* 前一个字符重复0次或任意多次
. 匹配除了换行符之外的任意一个字符
^ 匹配行首。 ^hello
+ 匹配前面的子表达式一次或多次
? 匹配前面的子表达式零次或一次，或指明一个非贪婪限定符
$ 匹配行尾。 hello$
[] 匹配括号中任意一个字符 [aevft]
[^] 取反,匹配除了括号内字符的任意一个字符 [^a-z]
\ 转义字符,用于将特殊字符的含义取消
\{n\} 其前面的字符恰好出现n次 [0-9]\{8\}
\{n,\} 其前面的字符出现不少于n次 [a-z]\{6,\}
\{n,m\} 其前面的字符出现n-m次 [^A-Z]\{2,6\}

实例分析:

grep "a*" regexp.learn
# 注意这里是匹配所有内容，而不是匹配所有包含a字母的行

grep "aaa*" regexp.learn
# 匹配至少含有2个连续a的行
 


grep "l..r"  regexp.learn
# 匹配l与r之间含有2个字符的行

grep "l.*r"  regexp.learn
# 匹配l与r之间含有任意字符

grep ".*" regexp.learn
# 依旧是匹配所有内容

grep -n "^$" regexp.learn
# 匹配空白行


grep "^[^0-9]" regexp.learn
# 匹配不以数字开头的行

grep "^[^a-zA-Z]" regexp.learn
# 匹配不以字母开头的行

grep "\.$" regexp.learn
# 匹配使用“.”结尾的行

grep "[0-9]\{3\}" regexp.learn
grep "[0-9]\{3,\}" regexp.learn
注意：匹配包含3个也包含大于3个，所以一般上面两种方法的含义一样

# **sed命令

sed [选项] '[动作]' 文件名

  • 描述：sed是一个包括在所有Unix平台(包括Linux)的轻量级编辑器。sed主要是用来将数据进行选取、替换、删除、新增，和vi不同的是，sed除了能对文件进行这些操作，更重要的是它还可以通过管道符对命令的结果进行编辑
  • 选项：
    • -n：只会把经过sed命令处理的行输出到屏幕，而不是把所有的数据都输出到屏幕
    • -e：允许对输入数据应用多条sed命令编辑
    • -i：用sed的修改结果直接修改读取数据的文件，而不是由屏幕输出
  • 动作：
    • a\：追加，在当前行后添加一行或多行。添加多行时，除最后一行外，每行末尾需要用“\”代表数据未完结
    • c\：行替换，用c后面的字符串替换原数据行。替换多行时，除最后一行外，每行末尾需要用“\”代表数据未完结
    • i\：插入，在当前行前插入一行或多行。插入多行时，除最后一行外，每行末尾需要用“\”代表数据未完结
    • d：删除，删除指定的行
    • p：打印，输出指定的行
    • s：字串替换，用一个字符串替换另外一个字符串。格式为“行范围s/旧字串/新字串/g”（类似于vim）
  • 示例：
[D-Cent@>/mnt9]$sed '2p' aa.txt 
ID	Name	PHP	Linux	MySQL	Average
1	Liming	82	95	86	87.66
1	Liming	82	95	86	87.66

> world' aa.txt

# 排序命令sort

sort [选项] 文件名

  • 选项
    • -f：忽略大小写
    • -n：以数值型进行排序，默认使用字符串型排序
    • -r：反向排序
    • -t：指定分隔符，默认分隔符是制表符
    • -k n[,m]：按照指定的字段范围排序。从第n字段开始，m字段结束（默认到行尾）
  • 示例：
sort /etc/passwd
sort -r /etc/passwd


sort -t ":" -k 3,3 /etc/passwd
sort -n -t ":" -k 3,3 /etc/passwd

# 按照文件类型进行判断

选项 作用
-b 判断文件存在与否，并且是否是块设备文件
-c 判断文件存在与否，并且是否是字符设备文件
-d 判断文件存在与否，并且是否是目录文件
-e 判断文件存在与否
-f 判断文件存在与否，并且是否是普通文件
-L 判断文件存在与否，并且是否是符号链接文件
-p 判断文件存在与否，并且是否是块管道文件
-s 判断文件存在与否，并且是否是非空
-S 判断文件存在与否，并且是否是套接字文件

若文件存在并且满足后面一项的条件，则返回值为真

# 按照文件权限进行判断

选项 作用
-r 判断文件是否存在，并且该文件是否有读权限
-w 判断文件是否存在，并且该文件是否有写权限
-x 判断文件是否存在，并且该文件是否有执行权限
-u 判断文件是否存在，并且该文件是否有SUID权限
-g 判断文件是否存在，并且该文件是否有SGID权限
-k 判断文件是否存在，并且该文件是否有SBit权限

[D-Cent@>/mnt28]$[ aa.txt -ef /mnt/hello ] && echo yes || echo noyes

-rw-r--r--. 2 root root 97 Sep 11 04:37 /mnt/hello

-ef 判断前者与后者的Inode号是否一致，可以理解为两个文件是否是一个文件，是用来判断硬链接的好方法

-eq 前者和后者是否相等
-ne 前者与后者是否不相等
-gt 前者是否大于后者
-lt 前者是否小于后者
-ge 前者是否大于等于后者
-le 前者是否小于等于后者
[D-Cent@>/mnt30]$[ 1999 -gt 55 ] && echo yes || echo no

# 多重条件判断

选项 作用

# 流程控制-until

  • 描述：until循环和while循环相反，until循环时只要条件判断式不成立则进行循环，并执行循环程序。一旦循环条件成立，则终止循环。
  • 语法：
until [ 条件判断 ]
do
    程序 
done
1
2
3
4
  • 示例：
#!/bin/bash 
# to get the sum of 1-100
# Author:blinderjay(blinderjay@gmail.com)

i=1
s=0
until [ $i -gt 100 ]
do 
    s=$(( $s+$i ))
    i=$(( $i+1 ))
done
echo "The sum is: $s"
